# 权限体系改造前端完整回归测试用例

日期：2026-07-01

## 1. 测试目标

本轮用于上线前完整前端回归，重点验证权限体系、工作空间模式、应用发布范围、智能体/工作流拆分、资产可见权限、组织后台管理和中英文文案是否闭环。

通过回归后，应能确认：

- 无工作空间成员、普通成员、空间管理员、空间负责人、组织管理员、组织所有者看到的入口和操作符合权限。
- 应用中心可见范围能覆盖成员、部门、工作空间、整个组织、无工作空间成员等场景。
- 工作空间权限模板、直接权限、空间管理员、空间负责人之间的展示和编辑逻辑清晰。
- 智能体和工作流在列表、详情、侧栏、创建入口、导入导出、日志、发布等入口上已经拆分。
- 知识库、数据库、文件等资产模块的查看、创建、编辑、删除和子能力权限不会越权。
- 权限编辑时，创建类、管理类、子功能类权限会自动补齐必要的查看权限，并有明确二次确认。
- 组织后台、前台工作空间管理、通讯录分配工作空间的角色和权限表现一致。
- 老模板、已删除模板、老发布数据、无工作空间成员等兼容场景不会出现空白选择器或不可恢复状态。
- 页面不出现连续错误提示、空白选择器、英文原始角色文案、按钮可见但点击失败等问题。

## 2. 执行策略

### 2.1 优先级说明

| 优先级 | 执行要求 | 说明 |
| --- | --- | --- |
| P0 | 必跑 | 主链路、权限边界、线上最容易影响用户的路径 |
| P1 | 建议完整执行 | 子功能、复杂组合、兼容场景、常见误操作 |
| P2 | 有时间执行 | i18n、视觉适配、极端状态、辅助检查 |

### 2.2 回归顺序建议

1. 先用组织所有者准备测试数据、成员、部门、工作空间和默认权限模板。
2. 先跑 P0 主链路，确认没有阻塞性问题。
3. 再按“角色边界、应用中心、工作空间管理、智能体、工作流、知识库、数据库、文件”的顺序跑 P1。
4. 最后切换英文环境、窄屏窗口、无痕窗口补充 P2。
5. 每次修改权限后，建议被测账号重新刷新页面，必要时重新登录，避免前端缓存干扰判断。

### 2.3 本文档口径

- 所有权限均以页面弹窗中能看到的中文权限文案为准。
- 测试同学不需要理解或填写研发权限码。
- 发现问题时，优先记录页面文案、操作步骤、账号角色、当前工作空间、截图和 Console/Network 错误。

| 智能体 | 智能体列表、智能体详情 | 列表过滤、详情路由、创建弹窗、发布、日志 |
| 工作流 | 工作流列表、工作流详情 | 列表过滤、详情路由、模板创建、导入导出、调试 |
| 知识库 | 知识库列表、知识库详情 | 查看知识库、查看知识文档、知识图谱、召回测试、创建/编辑/删除 |

不作为本轮前端完整回归重点：

- API Key 外部调用链路压测。
- 数据库迁移 SQL 逐行核对。
- 后端单元测试覆盖率。
- 大规模性能压测。

## 4. 环境、账号和资源准备

### 4.1 环境要求

| 项目 | 要求 |
| --- | --- |
| 前端地址 | 测试环境前端地址，例如本地 2880 端口或线上测试地址 |
| 后端地址 | 与前端配置一致，接口可正常访问 |
| 浏览器 | Chrome 最新稳定版优先 |
| 控制台 | 打开 DevTools Console 和 Network，观察 400、403、500、重复 toast、死循环请求 |
| 语言 | 至少跑中文；完整回归建议补跑英文，重点检查角色和权限文案 |
| 窗口尺寸 | 至少覆盖 1440px 桌面宽度和较窄桌面窗口 |

建议测试前清理浏览器缓存或使用无痕窗口，避免旧的工作空间上下文、本地存储、前端缓存干扰。

### 4.2 推荐账号矩阵

建议一个组织内至少准备两个工作空间：测试空间 A、测试空间 B。

| 账号代号 | 组织角色 | 工作空间状态 | 用途 |
| --- | --- | --- | --- |
| U1 | 组织所有者 | 可在 A/B 任意空间 | 验证组织后台、管理员、负责人转让、整个组织可见 |
| U2 | 组织管理员 | 可不加入任何空间 | 验证组织管理员天生具备管理工作空间和权限的能力 |
| U3 | 普通成员 | A 的空间负责人 | 验证空间负责人能力、成员管理、设置空间管理员 |
| U4 | 普通成员 | A 的空间管理员 | 验证空间管理员进入概览、管理成员、查看额度 |
| U5 | 普通成员 | A 的高级成员 | 验证大部分资产操作权限 |
| U6 | 普通成员 | A 的基础成员 | 验证基础开发权限和只读资源使用 |
| U7 | 普通成员 | A 的只读成员 | 验证只读入口、按钮隐藏 |
| U8 | 普通成员 | 无任何工作空间 | 验证个人工作台、应用中心、无空间状态 |
| U9 | 普通成员 | B 成员，不在 A | 验证跨工作空间可见范围 |
| U10 | 普通成员 | 使用老自定义模板或已删除模板 | 验证废弃模板提示和更换模板 |
| U11 | 普通成员 | A 中仅配置直接权限 | 验证不依赖模板的直接权限表现 |

### 4.3 推荐资源准备

| 资源 | 要求 |
| --- | --- |
| 智能体 | A/B 各 1 个，A 中至少 1 个已发布 WebApp 和应用中心 |
| 工作流 | A/B 各 1 个，A 中至少 1 个已发布 WebApp 和应用中心 |
| 知识库 | A 中 1 个，包含至少 2 个文档；如有知识图谱能力，开启 1 个图谱知识库 |
| 数据库 | A 中 1 个，包含至少 1 张表、若干记录 |
| 文件 | A 中 1 个普通文件、1 个文本文件、1 个文件夹 |
| 部门 | 至少 1 个父部门、1 个子部门，分别放入不同成员 |
| 额度 | 至少覆盖无限额度和有限额度两种工作空间 |
| 老数据 | 至少准备 1 个老自定义模板或已删除模板关联成员 |

### 5.2 资产可见权限组合

这组是本次需要特别回测的权限组合。测试时在“成员权限”弹窗中给测试账号设置对应权限。

| 权限组合 | 知识库预期 | 数据库预期 |
| --- | --- | --- |
| 不勾选任何知识库/数据库权限 | 不显示对应入口，或进入后显示无权限；列表不应反复报错 | 同左 |

| 勾选“查看数据库”和“查看表结构” | 不适用 | 可查看数据库表结构和字段信息，但不能查看表数据 |

### 5.4 默认模板行为

| 模板/角色 | 预期表现 |
| --- | --- |
| 空间负责人 | 固定角色，具备空间内完整管理能力；不能通过普通模板修改；只能通过转让或重新指派变更 |
| 空间管理员 | 固定角色，具备空间成员和权限管理能力；不展示下方细分权限列表；可由空间负责人或组织管理员设置/取消 |
| 高级成员 | 面向技术骨干，具备大部分资产创建、编辑、发布和内容管理权限，但不是空间管理员 |
| 基础成员 | 面向新人，至少具备智能体基础创建/编辑能力，并能查看必要资源；高危删除和团队管理入口不应开放 |
| 只读成员 | 面向审计或熟悉业务成员，可查看现有资产和使用/调试允许的能力；大多数创建、编辑、删除按钮不应展示 |
| 自定义模板 | 可创建、编辑、删除；应用给成员后形成权限快照，后续模板变化不应静默影响成员 |
| 旧模板或已删除模板 | 成员权限仍能正常展示；模板区域显示废弃或已删除提示，并支持更换模板 |

## 6. 完整回归用例

### 6.1 登录、全局导航和工作空间上下文

| ID | 优先级 | 前置条件 | 操作步骤 | 期望结果 |
| --- | --- | --- | --- | --- |
| NAV-01 | P0 | U8 无工作空间 | 1. U8 登录。2. 打开前台首页。3. 查看左侧导航和顶部工作空间状态。 | 标题为“概览面板”；没有当前工作空间时，不展示强依赖工作空间的资产管理入口；可进入应用中心；不出现连续 toast。 |
| NAV-02 | P0 | U5 属于 A | 1. U5 登录。2. 在工作空间切换器中切换 A/B 或无权限空间。 | 只能切换到自己可进入的空间；切换后导航和当前空间名称同步刷新。 |
| NAV-03 | P0 | 当前账号被移出当前工作空间 | 1. 管理员移除该账号。2. 被测账号刷新前台首页。 | 不连续请求当前权限并弹多次错误；页面回退到可用的无空间或组织状态。 |
| NAV-04 | P1 | U2 是组织管理员但不在任何空间 | 1. U2 登录前台。2. 查看导航和后台入口。 | 前台不会假装存在当前工作空间；组织后台入口可用；不出现无空间导致的空白页。 |
| NAV-05 | P1 | U5 同时属于 A/B | 1. 切换工作空间。2. 打开智能体、工作流、知识库、数据库、文件列表。 | 列表和按钮按当前空间权限刷新；不会残留上一个空间的数据或权限。 |
| NAV-06 | P2 | 较窄窗口 | 1. 缩小浏览器宽度。2. 打开侧栏、切换器、用户菜单。 | 文案不重叠，按钮不挤压，菜单能完整显示。 |

| ID | 优先级 | 前置条件 | 操作步骤 | 期望结果 |
| --- | --- | --- | --- | --- |
| OV-01 | P0 | U5 属于 A | 1. 打开概览面板。2. 查看资产数量、可用应用、最近工作。 | 数据按 U5 可访问范围聚合；不因切换工作空间导致概览整体变成另一个单空间看板。 |
| OV-02 | P0 | U4 是空间管理员 | 1. 打开概览面板。2. 查看右侧权限卡、额度卡。 | 权限模板显示“空间管理员”；组织角色使用中文文案；额度接口不报错。 |
| OV-03 | P0 | 有最近工作数据 | 1. 点击最近工作中的智能体、工作流、知识库、数据库、文件。 | 智能体进入智能体详情；工作流进入工作流详情；数据库进入数据库详情；知识库/文件进入正确详情或受权限保护。 |

### 6.3 应用中心和发布范围

| ID | 优先级 | 前置条件 | 操作步骤 | 期望结果 |
| --- | --- | --- | --- | --- |
| AC-01 | P0 | U8 无工作空间，应用显式发布给 U8 | 1. U1/U3 编辑某工作流应用中心范围，添加 U8。2. U8 登录应用中心。 | U8 能看到该应用并打开；不要求 U8 加入工作空间。 |
| AC-02 | P0 | 应用仅发布给 B | 1. U5 只在 A。2. U9 在 B。3. 分别打开应用中心。 | U5 不可见；U9 可见。 |
| AC-03 | P0 | U1 或 U2 操作 | 1. 打开发布设置。2. 将应用中心范围设置为“整个组织”。3. 保存。 | 所有组织成员可见；“整个组织”是独立状态，不作为普通可删除标签展示。 |
| AC-04 | P0 | 普通成员操作 | 1. U5 打开同一发布设置。2. 尝试设置“整个组织”。 | 入口不可用或保存被明确拦截；文案说明只有组织所有者/管理员可开启。 |
| AC-05 | P0 | 发布设置弹窗 | 1. 选择指定部门、成员、工作空间。2. 保存。3. 重新打开。 | 已选对象以标签展示；部门、成员、工作空间均可回显；所属工作空间默认可见且不重复展示。 |

## 9. 重点观察项

测试过程中优先记录以下问题：

- 权限弹窗出现空白选择器、双滚动、底部按钮不可见。
- Owner/Admin/normal 等原始角色值泄漏到页面。
- “空间负责人”“空间管理员”“组织所有者”“组织管理员”文案不统一。
- 按钮可见但点击后 403 或无权限。
- 无工作空间成员被加入应用可见范围后，应用中心仍不可见。
- 智能体和工作流仍跳到旧混合页面。
- 退出工作空间后连续弹错误 toast。
- 空间管理员不能查看额度或不能管理成员权限。
- “查看知识库”“查看数据库”没有作为独立可选项出现。
- 开启创建、导入、子功能权限时，未自动补齐必要查看权限。
- “整个组织可见”被错误展示成可删除普通标签。
- 数据库页面跳转到旧路径或页面不存在。
- 权限模板修改后，已应用成员被静默改变。
- 仅创建权限的成员能看到不该看到的既有资产标题、详情或配置。
- 工作空间管理员或组织管理员看到的设置页入口和实际可操作能力不一致。
